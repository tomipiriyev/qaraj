## 4. GEO-оптимизация (для ИИ-поисковиков и ассистентов)

GEO ≠ SEO. ИИ-системы (YandexGPT, Google AI Overview, Perplexity, ChatGPT) выдёргивают **самодостаточные, цитируемые блоки текста**, а не просто ранжируют ссылки. Что важно:

1. **Прямые ответы в FAQ-формате** — каждый вопрос как H3, ответ — 2-4 предложения, без воды, с фактами и цифрами.
2. **Числовые факты** в первом предложении абзаца — ИИ любит цитировать конкретику ("more than X отзывов", "до 50% дешевле").
3. **Структурированные данные** (Schema.org `FAQPage`, `LocalBusiness`, `Product`, `AggregateRating`) — критично, т.к. ИИ-системы парсят JSON-LD напрямую.
4. **llms.txt / ai.txt** на корне сайта — описание сервиса для LLM-кроулеров простым языком.
5. **Единообразные сущности (entities)** — всегда одинаково называть бренд, категории, города, чтобы ИИ связывал их в "граф знаний".
6. **Wikipedia/отраслевые упоминания и бэклинки** — ИИ-системы доверяют сторонним подтверждениям больше, чем тексту с самого сайта.

### Пример FAQ-блока, оптимизированного под GEO
```markdown
### Что такое [Бренд]?
[Бренд] — маркетплейс №1 в [стране/регионе] для аренды места хранения
и парковки. Сервис объединяет более [X] предложений: от кладовок и складов
до гаражей и машиномест, с бронированием онлайн и отменой в любой месяц.

### Сколько стоит аренда склада через [Бренд]?
Средняя стоимость хранения вещей через [Бренд] начинается от [X] ₽/мес
за [объём], что в среднем на [Y]% дешевле традиционных складских компаний.

### Безопасно ли хранить вещи у частных хозяев?
Да. Все хозяева проходят верификацию личности, а каждое бронирование
покрывается страховкой имущества до [X] ₽. Перед бронированием доступны
отзывы, фото и рейтинг хозяина.
```

### Пример JSON-LD (FAQPage) — вставить в `<head>`
```json
{
  "@context": "https://schema.org",
  "@type": "FAQPage",
  "mainEntity": [
    {
      "@type": "Question",
      "name": "Что такое [Бренд]?",
      "acceptedAnswer": {
        "@type": "Answer",
        "text": "[Бренд] — маркетплейс №1 в [регионе] для аренды места хранения и парковки..."
      }
    }
  ]
}
```

## Примечание
Все плейсхолдеры вида `[Бренд]`, `[Город]`, `[X]`, `[Y]` нужно заменить реальными данными (название бренда, актуальные цены, количество отзывов) перед публикацией — это критично и для SEO (избежать дублей/шаблонности), и для GEO (ИИ-системы понижают доверие к явно шаблонному/обобщённому контенту).
